🐳 Dockerfile / IaC Findings

Checkov — Missing HEALTHCHECK (both containers)

CKV_DOCKER_2 — No HEALTHCHECK instructionMedium
backend/dockerfile & frontend/dockerfile

Docker cannot automatically detect unhealthy containers without a HEALTHCHECK directive.

Fix — backend/dockerfile: 
HEALTHCHECK --interval=30s --timeout=5s --start-period=10s CMD wget -qO- http://localhost:5001/ || exit 1
Fix — frontend/dockerfile: 
HEALTHCHECK --interval=30s --timeout=5s --start-period=15s CMD wget -qO- http://localhost:3000/ || exit 1

Checkov — No Non-Root User (both containers)

CKV_DOCKER_3 — Container runs as rootMedium
backend/dockerfile & frontend/dockerfile

Running as root inside a container is a security risk if the process escapes the container namespace.

Fix — backend/dockerfile (add before CMD): 
RUN addgroup -S appgroup && adduser -S appuser -G appgroup
USER appuser
Fix — frontend/dockerfile (in final stage, before CMD): 
RUN addgroup -S appgroup && adduser -S appuser -G appgroup
USER appuser

DL3018 / DL3016 — Unpinned apk and npm install -gLow
frontend/dockerfile:7, 19

OpenAPI — swagger.json Missing Security Definitions

CKV_OPENAPI_1/4/5/6 — No security definitions in swagger.jsonMedium
backend/static/swagger.json

The Swagger/OpenAPI spec does not define securityDefinitions, making API documentation incomplete and potentially misleading about authentication requirements.

Fix: Add to swagger.json: 
{
  "securityDefinitions": {
    "sessionAuth": { "type": "apiKey", "in": "cookie", "name": "session" }
  },
  "security": [{ "sessionAuth": [] }]
}
